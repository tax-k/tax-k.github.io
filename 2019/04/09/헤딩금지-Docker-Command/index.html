[헤딩금지] Docker Command

헤딩금지
2019-04-09
Docker, cmd, tip

언젠가 한번은 헷갈릴

pull

docker pull <이미지 이름>:<태그> : Docker 이미지를 레포지토리로부터 받아옵니다.

images

docker images : Docker 이미지 리스트 출력해요.

ps

docker ps : Docker 컨테이너 리스트 출력해요. docker ps -a 는 stop 된 컨테이너도 보여줍니다.

run

docker run <옵션> <이미지 이름> <실행할 파일> : Docker 이미지를 컨테이너로 만들어 실행합니다.

세부 옵션들도 있는데 자세한건 나중에 추가하는걸로

amazonlinux 이미지 안의 /bin/bash를 실행합니다. 이미지 이름 대신 이미지 ID를 사용해도 됩니다.

-i (interactive), -t (Pseudo-tty) , -it(동시에 써도 됨) 옵션을 사용하면 실행된 Bash 에 입/출력을 할 수 있습니다.

그리고 –name 옵션으로 컨테이너의 name(alias)을 지정할 수 있습니다. 이름을 지정하지 않으면 Docker가 랜덤으로 막 지어요.

고급버전

stop

docker stop <컨테이너 이름> or <컨테이너 ID> : 컨테이너 스탑이요.

start

docker start <컨테이너 이름> or <컨테이너 ID> : 컨테이너 스탙요.

restart

docker restart <컨테이너 이름> or <컨테이너 ID> : 컨테이너 재시작요.

rm

docker rm <컨테이너 이름 or 컨테이너 ID> : 컨테이너 삭제요.

rmi

docker rmi <이미지 이름>:<태그> or <이미지 ID>:<태그> : 이미지 삭제요.

docker rmi amazonlinux 처럼 이미지 이름만 지정하면 태그가 달라도 amazonlinux 이름의 모든 이미지가 삭제되니까 조심조심
